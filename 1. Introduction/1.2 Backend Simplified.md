# BACKEND TLDR

Every website can be split into 2 parts which are frontend and backend. Frontend is what user see on their screen all that visual etc and backend is what saves and manages your data. For example if you are on Lazada or Shopee, the backend would store your order history, your profile, search result etc.

Say you are on Shopee right now and if you click on order, what actually happens in the background? So basically every computer that is connected to internet can send message across the internet to another computer, that is also connected to the internet. So to visualise this, Shopee have their computer somewhere in the world and our computer is going to send a message continuing our order to that shopee. In this scenario, our computer (the one sending message) is called the *client* and the Shopee computer receiving the message is the *server*. 

But before this happens, computer generally can't receive messages from the internet by default, we (aspiring backend dev) need to program them to be able to receive message. In order to do that we need **backend programming language**. Now almost every programming language has a feature that turns a computer into a server that allows it to receive message. These are one of the popular backend programming language:

+ JavaScript or sometimes called as NodeJs
+ Python
+ Ruby
+ Java

However, using programming by itself is actually kinda hard and it require a huge amount of code, so there are 2 tools that we use to help with this:

1) backend framework
2) packet manager

A backend framework help us build server much easier and with a lot less code. Each programming language has a few different frameworks to choose from but the popular ones are :

+ ExpressJS for JavaScript
+ Python (Django)
+ Ruby (rails)
+ Java Spring.

In backend we also use a lot of code that other people have written called *packages* to do common tasks like calculation, talking to database, set up user login and etc. Backend Dev typically use a lot of packages in developing backend and in order to install and manage all these packages, we use package manager. Again, there are a fair few of package manager but these are the popular ones:

+ npm (JS)
+ pip (Python)
+ Bundler (ruby)
+ Maven (Java)

**These are all the technologies we need to build our backend server**
